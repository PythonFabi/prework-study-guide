Prework Study Guide

HTML

  • HTML(Hypertext Markup Language) gives the content of the webpage a structure.
  • Before the html element we have the declaration !DOCTYPE html to show browser type of doc.
  • html element can include attribute lang="en"(english), HTML attributes define additional info to element
  • The head element contains information about the webpage (for the browser).
  • title: browser title bar, style: CSS style for webpage, base: base url for webpage, link: links external resource to webpage
  • additional metadata like meta:character set, description, keywords, author, viewport etc.
  • e.g.: charset: UTF-8 (Unicode 8 defines that all types of characters can be used), viewport: lets website adjust to device
  • script: embeds client-side scripts in HTML documents
  • The body element represents the visible content shown to the user.
  • Both elements are parent elements, which have child elements nested inside
  • elements usually consist out of opening and closing tags, which have mostly visible content in between them
  • Some elements like br(line break) or img(for images) don`t have a closing tag
  • Elements like img and a(anchor) need attributes like src(image source) and href(link URL) for the browser to find them
  • More attributes like for img: alt(description if img doesn`t load and accessibility), height and width(img size)
  • Or id to identify elements and select them
  • Semantic HTML elements make webpage accessible for widest variety of users, also those with assistive devices
  • Use especially semantic elements with visible content
  • headings(h1(biggest)-h6) and paragraphs(p) semantic text elements(use h1 just once and in order to h6 for better accessibility)
  • ul, ol, and li for unordered list(bulleted), ordered list (numbered) and list items
  • Elements like div(creates divisions of webpage) and span(create inline division) are non-semantic
  • We have three main semantic elements in the body element, which are header, main and footer
  • Those can be divided into sections

CSS

  • CSS (Cascading style sheets) gives the content of a webpage style with (colors, aesthetics, visual layout and organization)
    and defines style rules for specific elements of an HTML page and format them using wide range of style options
  • 3 ways to use CSS: Inline CSS(use style attribute on specific HTML element), internal style sheet(embeds style element in head element one change in CSS rule applies to all tagged page elements), external style sheet(preferred in most cases, allows developers to work on seperate file)
  • external stylesheet will be linked with link element and attributes rel="stylesheet" (relation between HTML and external CSS sheet) and
    href="URL" (seperation of concerns)
  • CSS rules consist of: selector(element to style), and Declaration block(in curled braces with unlimited declarations)
  • Declarations consist of two elements: Property(e.g. color or text-align) and Value (each property holds a value, e.g. blue, center)
  • CSS classes and IDs: identify elements, class allow us to share CSS rule to any chosen element (.class as selector)
  • one declaration block can have several selectors, DRY(Do not repeat yourself, reuse and share pieces of code)
  • text-align: Right, Center, Left, justify
  • font-size, font-color, font-family, font-weight, font-style etc.: examples 20px, blue, Helvetica, bold, italic
  • display: block (each element start on new line), inline (all elements in the same line)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • An element can have a border around it, with the size value, with solid it becomes visible and with color you can determine the color
  • colors can be either just blue, grey etc. or it can have Hex-codes which begin with # and a combination of numbers and letters with 6 characters
  • 3 primary Hex colors are Red(#FF0000), Green(#00FF00) and Blue(#0000FF)
  • CSS Box Model: every HTML essentially a box, starts with margin outside, border, padding and content of element
  • Box-shadow: x-axis(to the right), y-axis(to the left) and color

Git

  • Git is a version control tool that let`s us revert back to different versions of our code without breaking our main code
  • Github is a cloud-based website that let`s us collaborate with other developers using version control with Git
  • A branch is a copy or version of code that can be worked on independently in isolation without overwriting our team members code
  • main(master) branch is our default version of code, which can be updated with feature- or development-branches, which are copies of default branch
  • Those can be merged back into the main branch, after change is completed
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all modifications in current working directory to the staging area of Git ready for commit
  • git commit –m [commit message]: descriptive commit message in imperative; max 50 characters, should be less; snapshot of currently staged changes
  • git pull origin main: look if local base branch is in sync with Github base branch, make sure to have latest version of project on local computer, not outdated
    (if a team member pushes changes to main branch, we always want to have the most updated version in our local branch to prevent sync issues)
  • git push origin [branch-name]: (if feature-branch name: feature/branch-name) push changes from local to remote Git branch
  • Visit GitHub website to compare and create pull request (PR), click green button
  • Descriptive PR title, instructions for team members reviewing on changes made, how feature is supposed to function and how to start app so feature can be tested
  • Designate base branch as main, and comparison branch as local (feature) branch, because local branch will merge into main
  • Create PR
  • Usually request one or two team members to view PR, after that “Merge PR”
  • Update local main branch with remote branch with: git pull origin main
  • Delete feature-branch after to prevent accidently reusing old branches: git branch -D [branch-name]
  • This is the usual GitHub workflow
